## 研究题目

*Privacy-Preserving AD/ADRD Phenotyping from EHR: A Three-Tier Benchmark of Edge, Cloud, and Frontier Language Models on Consumer Hardware*

核心问题：消费级笔记本上的轻量 LLM，能否在保护隐私的前提下，达到与云端 API 相近的 AD/ADRD 识别性能？

---

## 三路推理架构

| Tier | 机器 | 模型 | 隐私 | 成本 |
|------|------|------|------|------|
| Tier 0 | 任意 | ICD-10 规则 baseline | ✅ 本地 | $0 |
| Tier 1 | M4 MacBook | qwen2.5:1.5b via Ollama | ✅ 数据不出设备 | $0 |
| Tier 2 | M1 Pro | qwen-plus via DashScope API | ⚠️ 阿里云 | token 计费 |
| Tier 3 | M1（Claude Code） | claude-sonnet-4-6 直接读 CSV 分类 | ⚠️ Anthropic | 订阅制 |
| Agent | M4 | LangGraph 3-agent fan-out + qwen2.5:1.5b | ✅ 本地 | $0 |

---

## 关键架构决策

**为什么用 qwen2.5:1.5b 而不是 7b**
- 组员机器全是 M1，7b 跑不动；M4 可以跑 1.5b，精度接近 7b，0.5b 太弱
- 对应 AI infra 叙事：用最小模型、最普通硬件处理医疗数据

**为什么用 Qwen API 而不是 OpenAI**
- Qwen API 有免费 token（DashScope），DASHSCOPE_API_KEY 环境变量
- base_url: https://dashscope.aliyuncs.com/compatible-mode/v1，用 openai SDK 兼容调用

**为什么 Tier 3 用 Claude Code 而不是 API**
- 无需 API key，无需改代码，Claude Code 直接读 CSV 逐条分类写入结果
- 代表"交互式 AI 助手作为标注者"的新推理范式，差异化创新点

**数据隐私叙事**
- 教授明确说医疗数据不适合上云，临床更信任本地
- 三路对比的核心就是量化 privacy-performance-cost 三角 trade-off

## 代码结构

**主文件：** `hinf5026_final_project.py`（8 个模块）
- Module 1: `extract_relevant_text()` — EHR 文本预处理
- Module 2: `check_kappa()` — Cohen's Kappa
- Module 3: `evaluate()` — AUC/Precision/Recall/F1
- Module 4: `call_llm()`, `run_batch_inference()` — provider="ollama"|"qwen"
- Module 5-6: LangGraph Agent（icd/med/note 并行 fan-out + synthesis）
- Module 7: `run_agent_batch()` — 批量 Agent 推理
- Module 8: `compare_all_models()` — 对比可视化
