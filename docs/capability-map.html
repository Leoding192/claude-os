Claude OS Capability Map

All capabilities and how to invoke them in a new session.

Section 1 Commands

Direct commands for planning, execution, review, communication, and memory operations.

/plan <task>

Decompose task into structured plan.

/plan fix the auth bug
/write <topic>

Writing pipeline: draft to Codex blind review to revised output.

/write essay about AI design
/task <intent>

Start state-tracked task (PLANNING to EXECUTING to COMPLETED).

/task refactor the API layer
/review [target]

Code review: choose Claude or Codex engine.

/review or /review src/api.ts
/brief [date]

Daily brief: calendar events + email summary.

/brief or /brief 2026-03-18
/capture text

Quick-capture thought/task into session memory.

/capture fix the login bug later
/remember thing

Persist decision or lesson to long-term memory.

/remember always use zod for validation
/draft-email desc

Draft email with writing preferences applied.

/draft-email follow up with team on Q2
/adversarial-review [target]

Dual-engine review: Claude + Codex side-by-side.

/adversarial-review
/undo-last

Undo the most recent Reversible-class action.

/undo-last
/consolidate

Archive stale L3 memory entries.

/consolidate

Section 2 Skills

Reusable workflows triggered by natural-language requests or direct command usage.

debug

Systematic debugging: reproduce, isolate, fix root cause.

trigger: debug this, whats wrong with X
refactor

Restructure code for clarity without changing behavior.

trigger: refactor this, clean up
review

Structured code review workflow.

trigger: review this code
brief

Daily brief: calendar + email.

trigger: /brief or auto via launchd at 08:30
docx

Read/write/edit Word documents.

trigger: create a Word doc, edit this .docx
pdf

Read, fill forms, annotate PDFs.

trigger: read this PDF, fill the form
pptx

Create and edit PowerPoint presentations.

trigger: make a PPT, create slides
xlsx

Read/write/recalculate Excel spreadsheets.

trigger: create a spreadsheet, edit Excel

Section 3 Agents

Focused roles that activate based on task complexity, scope, and workflow state.

planner

Decomposes complex tasks into executable plans.

trigger: 3+ steps, architectural decisions
coder

Implements scoped tasks only after plan approved.

trigger: plan exists, scope locked
reviewer

Code review: correctness, security, style.

trigger: /review, /adversarial-review
documenter

Writes and updates technical documentation.

trigger: document this, docs missing
mailer

Gmail: read, search, draft, send emails.

trigger: email, draft, reply, inbox
scheduler

Apple Calendar: read events, create reminders.

trigger: calendar, meeting, reminder, agenda

Section 4 MCP Servers

Tool backends that provide repo, mail, calendar, notification, and filesystem integration.

github

Repos, issues, PRs, code search.

tools: search_repositories, create_issue, get_file_contents
gmail

Search, read threads, create drafts.

tools: gmail_search_messages, gmail_read_message, gmail_create_draft
apple-calendar

Read/create/update calendar events.

via: osascript AppleScript
notifications

Send macOS system notifications.

via: osascript display notification
filesystem

Read/write local files.

scope: ~/claude-os and ~/projects

Section 5 Hooks

Auto-execute behaviors with no manual invocation required.

PreToolUse

Block any edit on main/master branch.

auto-execute, no manual invocation
PostToolUse format

Auto-format files: prettier / black / gofmt / shfmt.

auto-execute, no manual invocation
PostToolUse audit

Log Confirm-tier Bash ops to logs/audit.jsonl.

auto-execute, no manual invocation
UserPromptSubmit

Inject memory/session.md at every session start.

auto-execute, no manual invocation
Stop

Auto-clear session.md, carry forward incomplete tasks.

auto-execute, no manual invocation

Section 6 Raycast

Quick-launch entry points for briefings, capture, and new terminal sessions.

Claude Brief

Runs /brief, shows output in Raycast.

Cmd+Shift+B
Claude Capture

Quick-capture with text input field.

Cmd+Shift+Space
Claude Ask

Opens new Claude session in Terminal.

Cmd+Shift+A

Section 7 Memory

Layered memory model spanning session context, project context, durable knowledge, and archives.

L1 memory/session.md

Auto-injected every session, limit 50 lines.

write: /capture or natural language
L2 memory/projects/name.md

Per-project decisions, on demand.

write: /remember
L3 decisions.md, people/, writing.md

Durable knowledge, on demand.

write: /remember
L4 memory/archive/YYYY-MM/

Archived stale entries, never auto-injected.

write: /consolidate
