模型分配策略

Memory

4 layers · ~/claude-os/memory/
L1
Session State
memory/session.md
当前工作状态：In Progress、Blocked、Next Up、Captures。每次 session 自动注入。<50 行，只记 bullets。
✅ 自动注入
L2
Project Context
memory/projects/<name>.md
每个项目一个文件。当前目标、架构决策（为什么选 X）、已知 tradeoff、集成点。现有：hinf5026.md。
按需读取
L3
Durable Decisions
memory/decisions.md
memory/people/
memory/writing.md
持久决策 + 教训。/remember 写入。decisions.md 有 Decisions 表格 + Lessons Learned。writing.md 记录写作偏好。
按需读取
L4
Archive
memory/archive/YYYY-MM/
L3 陈旧条目的归档层。/consolidate 命令触发。保留 2 年。永不自动注入。
月度 /consolidate
写入规则说明
/remember唯一正确的 L3 写入方式。不能跳过 /remember 直接写 decisions.md。
/capture快速写入 L1 session.md，适合不打断 flow 的临时记录。
不要自动晋级L1→L3 需要显式 /remember 或"记住"信号，不能自动 promote。
L3 写入格式rule/fact → Why: (原因) → How to apply: (适用场景)。结构化，不写流水账。
/consolidate将 L3 中超过 30 天的陈旧条目迁移到 L4 archive。建议月度执行。

MCP Integrations

~/claude-os/.mcp.json
github
npx server
官方 GitHub MCP。读写 repo、issues、PR、branches、files、搜索。需设置 GITHUB_PERSONAL_ACCESS_TOKEN 环境变量。
→ "提交 PR 到 claude-os" / "查看 open issues" / "创建 feature branch"
gmail
Claude.ai 内置
Claude.ai 内置 Gmail 集成，无需本地服务。搜索邮件、读取线程、创建草稿、列标签。发送需 Confirm 权限。
→ "搜索来自导师的邮件" / "起草进度报告" / "列出未读邮件"
apple-calendar
Bash/AppleScript
无独立服务器。cal-manager agent 通过 osascript 命令操作 Calendar.app。需授权 Terminal → Automation。
→ "明天有什么会议？" / "帮我安排下周一的会议" / "今天的日程"
figma
Remote HTTP
官方 Figma MCP，远程 HTTP 连接 mcp.figma.com/mcp。读取设计文件、frames、组件、样式、变量。Free plan: 6 calls/month。
→ "把这个 Figma frame 转成 React 组件" / "获取设计 token"
filesystem
npx server
本地文件系统读写。允许访问目录：~/claude-os 和 ~/projects。读取、写入、移动、搜索文件。
→ 直接文件操作（通常 Claude 自身工具已够，MCP 提供额外访问路径）
notifications
Bash/osascript
macOS 系统通知，无独立服务器。通过 osascript -e 'display notification...'。/brief 完成时发通知。
→ 任务完成提醒 / /brief 推送 / Layer 4 定时任务提醒
ℹ️
computer-use 和 Telegram 通过 Plugins 接入

Computer use（截图、鼠标键盘）和 Telegram（机器人消息）不在 .mcp.json 里，它们是 Claude Code Plugins，见下方 Plugins 章节。

Hooks

6 events · ~/claude-os/settings.json
SessionStart
matcher: any
会话启动时
注入当前日期 + memory/session.md 内容 + 最近 git log，让 Claude 立即了解工作状态。
PreToolUse
Edit|Write|MultiEdit|NotebookEdit
文件编辑前
检测当前 git branch，若为 main/master 则 BLOCK 并提示先创建 feature branch。
PreToolUse
Edit|Write|MultiEdit
文件编辑前
检查文件路径，若包含 .env / secrets / credentials / package-lock.json 等则 BLOCK。
PostToolUse
Edit|Write|MultiEdit
文件编辑后
按扩展名自动格式化：.js/.ts/.json/.html/.md → prettier；.py → black；.go → gofmt；.sh → shfmt。formatter 缺失时发警告。
PostToolUse
Edit|Write|MultiEdit
文件编辑后
如果 .ts/.tsx/.js/.jsx 文件存在同名 .test.* 文件，自动运行 npx jest，输出最后 8 行失败信息。
PostToolUse
Bash
Bash 执行后
Confirm 级别操作（git push / git reset --hard / rm -rf / send / delete）写入 ~/claude-os/logs/audit.jsonl 审计日志。
UserPromptSubmit
matcher: any
每次提交 prompt
注入 memory/session.md 内容到上下文，确保 Claude 始终知道当前任务状态。
Stop
matcher: any (×4)
Claude 回复结束后
① 更新 session.md 提醒；② 追踪任务状态文件；③ 提醒 /remember；④ 运行 hooks/auto-memory.py 自动提取关键信息。
PreCompact
matcher: any
上下文压缩前
重新注入 session context（日期 + session.md），防止压缩后丢失工作状态。

Plugins

3 enabled · ~/.claude/settings.json

Claude Code 插件，独立于 MCP。通过 Claude Code 插件市场安装，持久运行。

claude-hud
jarrodwatts/claude-hud · v0.0.10
状态栏（statusLine）插件。在终端底部实时显示 Claude 工作状态、当前任务、token 使用情况。由 Node.js 进程运行。
telegram
claude-plugins-official
Telegram Bot 集成。接收 Telegram 消息后可触发 Claude，通过 reply 工具回复。支持图片附件、消息反应。不能查看历史消息。

OpenAI Codex CLI 集成插件。配合 codex / codex-bridge skills 使用，实现 Claude + Codex 双引擎 review（/adversarial-review）。

Commands

13 slash commands · ~/.claude/commands/
/plan <task>
调用 task-planner 分解任务，生成计划后等待批准再执行。
触发：范围不清、3+ 步骤、架构影响
/task <intent>
开启状态追踪任务（PLANNING → EXECUTING → COMPLETED），跨轮次保持进度。
触发：需要持久化任务状态时
/review [target]
代码审查，可选 Claude 或 Codex 引擎。传入文件路径、diff 或最近改动。
触发：合并前、质量检查
/adversarial-review
双引擎盲审：Claude code-reviewer agent + Codex 各自独立 review，并排比较结果。
触发：高风险代码、架构决策
/simplify [target]
检查代码复用性、质量、效率，自动修复发现的问题。
触发：实现完成后，提交前
/remember <thing>
将决策或教训持久化写入 memory/decisions.md（L3）。
触发：有价值的经验、需跨 session 保留的规则
/capture "<text>"
快速将想法 / 任务记录到 memory/session.md（L1），不打断工作流。
触发：随手记，临时 note
/consolidate
归档 L3 中陈旧的 memory 条目到 memory/archive/（L4）。
触发：月度清理（建议每月一次）
/brief [date]
日报：今日 Calendar 日程 + Gmail 邮件摘要。08:30 via launchd 也会自动运行。
触发：早晨启动，或需要即时日报
/morning
早间资讯：调用 news-curator，获取 AI / 数据 / 金融行业最新动态 + 任务优先级确认。
触发：每天早晨
/night
晚间收尾：调用 session-summarizer，总结今日进度 + 生成明日计划 + 更新 session.md。
触发：每天结束工作前
/draft-email <desc>
调用 mail-writer agent，以 Leo 的写作偏好起草邮件。
触发：写重要邮件
/write <topic>
写作流水线：Claude 起草 → Codex 盲审 → Claude 修订 → 最终输出。
触发：文档、博客、正式写作
已移除 / 合并 /undo-last — 文件存在但作为 reference；/teach-impeccable 和 /frontend-design 已合并到 /impeccable。
